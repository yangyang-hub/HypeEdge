#### 2.5.1 做市策略工作台 `/strategy/[id]/market-making`

做市不能复用趋势策略的弱类型参数卡片。前端必须按 `strategy_type` 使用 discriminated union，并提供：

- **Overview**：STOPPED/WARMING/SHADOW/RUNNING/PAUSED/DRAINING/FAULTED、数据新鲜度、quote uptime 和告警。
- **Live Quotes**：盘口、fair/reservation、desired/live/inflight/UNKNOWN quote slots、revision 和 quote age。
- **Inventory & Skew**：库存 soft/hard/emergency bands、skew、margin、liquidation distance 和 funding carry。
- **PnL**：分开显示与 Postgres ledger 对账的 Accounting PnL、Execution Quality/Markout 和 Inventory Episodes，
  不把 markout 重复计入净 PnL。
- **Action Budget**：remaining credits、burn/earn、USDC/action、runway、emergency reserve 和预算状态。
- **Configuration**：强类型表单、版本 diff、shadow 预览、审批、激活和回滚。
- **Events**：risk、reconciliation、execution UNKNOWN、生命周期和 operator audit。

控制 API 使用 `/api/v1/strategies/{id}` 与 `/api/v1/market-making/{id}`；所有 mutation 带 `Idempotency-Key`，
配置激活带 revision/`If-Match`。可靠状态通过 durable SSE 更新，高频 fair/quote/inventory 通过有界 WebSocket 更新。
首次加载/重连必须通过 REST 获取 Postgres 权威 position/runtime/action-budget，WebSocket 只做显示优化并携带 revision；
gap/stale 时全量 resync。所有卡片显示最后更新时间和 stale 状态，mainnet 危险操作使用二阶段确认并显著标识环境。

## 3. 实时数据流架构

```
                  ┌──────────────────┐
                  │  Hyperliquid WS  │
                  │  wss://api.../ws │
                  └────────┬─────────┘
                           │ l2Book / trades / candle / allMids
                           ▼
              ┌────────────────────────┐
              │   前端 WebSocket Hook  │  ← 直连，不经过后端
              │   useHyperliquidWS()   │     (降低延迟)
              └────────┬───────────────┘
                       │
                       ▼
              ┌────────────────────┐
              │  Zustand Store     │  ← 全局行情状态
              │  (marketStore)     │
              └────────┬───────────┘
                       │
          ┌────────────┼────────────┐
          ▼            ▼            ▼
     K线图表组件    盘口组件     成交列表组件


              ┌────────────────────────┐
              │  后端 FastAPI (SSE)     │
              │  /api/events           │
              └────────┬───────────────┘
                       │ SSE: 账户/持仓/订单/策略/风控变更
                       ▼
              ┌────────────────────┐
              │  SWR + SSE Hook   │  ← 订阅后端事件
              │  useAccountSSE()  │
              └────────┬───────────┘
                       │
          ┌────────────┼────────────┐
          ▼            ▼            ▼
     账户卡片组件   持仓表组件    订单表组件

低频数据 (策略状态、风控面板):
  SWR 轮询 5s → GET /api/strategies, /api/risk/status
```

**数据获取策略总结：**

| 数据类型 | 获取方式 | 刷新频率 | 延迟要求 |
|---------|---------|---------|---------|
| 盘口/成交/K线 | 前端直连 HL WS | 实时 | < 1s |
| 账户/持仓/订单 | 后端 SSE 推送 | 实时 | < 2s |
| 策略状态 | SWR 轮询 | 5s | 不敏感 |
| 风控状态 | SWR 轮询 | 5s | 不敏感 |
| 权益曲线 | SWR 轮询 | 60s | 不敏感 |
| K线历史 | REST API | 一次性 | 不敏感 |

## 4. 告警与通知

**页面内告警（顶部 Toast）：**
- 订单被拒绝
- 策略停止
- 对账失败

**全屏告警横幅：**
- Kill Switch 触发 → 红色全屏横幅，不可关闭，直到重置
- 连接断开超过 30s → 黄色横幅

**浏览器通知（可选）：**
- Kill Switch 触发
- 大额亏损（日亏损 > 2%）
